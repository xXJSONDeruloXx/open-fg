## Root cause of the title mismatch

The GRIS log showing `app=SB-Win64-Shipping.exe` was a **cross-game contamination artifact**:
- Stellar Blade (`SB.exe`) was still running from a previous probe when the GRIS probe started
- The GRIS probe's `cleanup_matches` function only kills processes matching `AppId=683320` and `GRIS.exe`
- It did **not** kill the still-running Stellar Blade process
- The OMFG layer instance inside the Stellar Blade process was therefore still writing to its log file
- The GRIS probe's log file (a unique path set per-run by `OMFG_LAYER_LOG_FILE`) was _different_ from the Stellar Blade log path — but since the wrapper env was overwritten for GRIS, the captured file may have been the stale Stellar Blade wrapper log rather than a fresh GRIS log

## Harness fix

`scripts/test-steamdeck-steam-game.sh` now includes a `kill_all_steam_games()` function that:
- runs **before** `cleanup_matches` at the start of each probe
- kills all processes matching any `AppId=\d+` pattern, `SteamLaunch`, `waitforexitandrun`, `reaper steam`, `wineserver`, `wine64`, or `wine ` markers
- waits 3 seconds after the broad kill before the game-specific `cleanup_matches`

Syntax validated: `bash -n scripts/test-steamdeck-steam-game.sh`
